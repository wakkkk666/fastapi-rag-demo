## 检索相似度显示（第三优先级）✅

**原理**：ChromaDB 默认使用 `l2`(平方欧氏距离)。SentenceTransformer 输出归一化向量，因此：
- `similarity = max(0, 1 - L2_distance/2) * 100`

**后端改动**：
- 每条 source 新增 `similarity` 字段（百分比，保留1位小数）
- 验证：distance=0 → 100%, distance=1.35 → 32.3%, distance=2 → 0%

**前端改动**：
- 来源卡片头部从「🟢高相关 d=1.0146」→ 「32.3%」醒目百分比徽章
- 颜色分级改为基于相似度：≥70% 绿 / ≥40% 黄 / <40% 红
- CSS：`.src-similarity` 替代 `.src-distance`，圆形药丸徽章

## 多知识库联合检索升级 ✅

**后端唯一改动**：`/chat` QA 模式检索逻辑

```
之前：collection.query(where={"collection": {"$in": [...]}}, n_results=3)
      → 单次全局查询，取 Top3，大知识库可能挤掉小知识库结果

现在：for each collection → .query(where={"collection": cname}, n_results=3)
      → 每个知识库独立 Top3 → 合并 → 按 distance 排序 → 取 Top10
```

**前端无改动**：复选框多选 + `collection_names` 数组发送已有。

**验证结果**：
- 2个知识库 × 3 = 6个来源
- 两个库的结果都有覆盖（简历 3条 + JD 3条）
- 按相似度降序：48.7% → 45.7% → 33.0% → 26.5% → 22.5% → 21.4%
- Context 带 `[来源: xxx.pdf]` 标注

**根因**：`marked` CDN 同步加载可能超时/失败 → 阻塞页面的 inline `<script>` 执行 → `showPage` 未定义。

**修复**：
- 移除 `marked.js` CDN 依赖，用自写 `renderMd()` 替代
- 所有 onclick 回调函数挂到 `window` 上：`window.showPage = function...`
- `"use strict"` 严格模式
- `let`/`const`/`forEach`/`for...of` 全部改为 `var` + `for(;;)` 循环（最大兼容）
- `String.includes()` 改为 `indexOf()`

**问题**：`querySelector('.nav-item[onclick*="..."]')` 在浏览器中引号转义导致返回 null，showPage 报错中断。

**修复**：
- 导航项增加 `data-page="home"` 等属性
- 改用 `.nav-item[data-page="name"]` 选择器
- showPage 用 for 循环替代 forEach（兼容性更好）
- 增加 `console.log` 和 `console.error` 调试日志
- `window.onload` 显式调用 `showPage('home')`
- 取简历和JD两份PDF全文（summary模式）
- 拼接结构化Prompt（7维度分析）
- DeepSeek一次调用输出：综合匹配度/技能匹配/缺失技能/项目分析/竞争力/技能补齐/通过率预测
